# Design Thinking for HostelHub

## 1. Empathize
- **Users:** Students, wardens, maintenance staff
- **Pain Points:**
  - Students: Slow reporting, no updates, confusion
  - Wardens/Staff: Delayed info, hard to track issues

## 2. Define
- **UI Problem:**
  - Hostel residents need a fast, clear way to report issues and see updates.
  - Staff need to manage and resolve complaints easily.

## 3. Ideate
- **Screens:**
  - Login
  - Dashboard/Home
  - Issue Reporting Form
  - Issue List & Status
- **Layout:**
  - Simple navigation, big buttons, clear forms

## 4. Prototype
- **Figma:**
  - Draw each screen with boxes for buttons, fields, lists
  - Arrange UI components for clarity

## 5. Test
- **Flutter:**
  - Build screens, check if they match Figma
  - Get feedback, refine UI

**Each stage helps make HostelHub easy and useful for everyone.**
